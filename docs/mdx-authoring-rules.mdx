# MDX 写作规则

本项目的 MDX 内容用于本地预览工具渲染。为保证稳定，AI 生成 MDX 时必须遵守本规则。

## 总原则

- 只写内容，不写业务逻辑。
- 禁止在 MDX 中写 `import` / `export`。
- 禁止定义自定义 React 组件。
- 禁止使用未列入白名单的 JSX 组件。
- 禁止执行任意 JS，例如 `{fetch(...)}`、`{window...}`、复杂表达式等。
- 可使用 Markdown 标准语法：标题、段落、列表、引用、表格、链接、图片、代码块。
- 可使用本文列出的组件。

## Frontmatter

每个 MDX 文件建议包含：

```mdx
---
title: 文档标题
description: 简短描述
---
```

规则：

- `title`：必填，短标题。
- `description`：可选，一句话摘要。
- 不要在 frontmatter 中写复杂对象，除非后续规则明确允许。

## Markdown 基础语法

允许：

```mdx
# 一级标题

## 二级标题

正文内容。

- 列表项 A
- 列表项 B

> 引用内容

[链接文字](https://example.com)

![图片描述](./image.png)

| 列 A | 列 B |
| --- | --- |
| 1 | 2 |
```

## 代码块

允许 fenced code block：

````mdx
```ts
function hello(name: string) {
  return `Hello ${name}`
}
```
````

规则：

- 代码块只展示代码，不执行。
- 必须标注语言，例如 `ts`、`tsx`、`js`、`bash`、`json`、`mdx`。

## 组件白名单

当前允许使用以下组件：

### Callout

用于提示、警告、说明。

```mdx
<Callout>
这是一个提示。
</Callout>
```

可选写法：

```mdx
<Callout title="注意">
这里是说明内容。
</Callout>
```

规则：

- 用于短提示，不要放过长正文。
- 不要嵌套复杂组件。

### Cards / Card

用于展示入口、链接、资源卡片。

```mdx
<Cards>
  <Card title="快速开始" href="https://example.com">
    阅读快速开始指南。
  </Card>
  <Card title="API 文档" href="https://example.com/api">
    查看 API 用法。
  </Card>
</Cards>
```

规则：

- `Card` 必须有 `title`。
- `href` 可选。
- `Card` 必须放在 `Cards` 内。

### Tabs / Tab

用于普通内容标签页，例如不同平台、不同方案。

```mdx
<Tabs items={["pnpm", "npm"]}>
  <Tab>

```bash
pnpm install
```

  </Tab>
  <Tab>

```bash
npm install
```

  </Tab>
</Tabs>
```

规则：

- `Tabs` 推荐使用 `items` 声明标签名称。
- `Tab` 顺序必须和 `items` 对应。
- 如果不用 `items`，每个 `Tab` 必须写 `value`。

### Accordions / Accordion

用于 FAQ、折叠说明。

```mdx
<Accordions>
  <Accordion title="为什么打不开？">
    请确认文件后缀是 `.mdx` 或 `.md`。
  </Accordion>
  <Accordion title="是否支持本地图片？">
    后续会支持相对路径资源解析。
  </Accordion>
</Accordions>
```

规则：

- `Accordion` 必须有 `title`。
- `Accordion` 必须放在 `Accordions` 内。
- 内容不要嵌套过深。

### Steps / Step

用于流程、教程、分步说明。

```mdx
<Steps>
  <Step>
    第一步：打开文件。
  </Step>
  <Step>
    第二步：查看预览。
  </Step>
</Steps>
```

规则：

- `Step` 必须放在 `Steps` 内。
- 每个 `Step` 表达一个清晰动作。

### Files / File / Folder

用于文件列表或目录结构。

```mdx
<Files>
  <Folder name="src" defaultOpen>
    <File name="App.tsx" />
    <File name="components/MdxPreview.tsx" />
  </Folder>
  <File name="package.json" />
</Files>
```

规则：

- `File` 必须有 `name`。
- `Folder` 必须有 `name`。
- `File` / `Folder` 必须放在 `Files` 内。

### Banner

用于顶部横幅提示。

```mdx
<Banner variant="normal" changeLayout={false}>
  新版本已发布。
</Banner>
```

规则：

- 本地文档中必须写 `changeLayout={false}`，避免影响应用布局。
- `variant` 可用 `normal` 或 `rainbow`。
- 不要写 `id`，除非需要可关闭并记忆状态。

### Heading

用于显式控制标题级别。

```mdx
<Heading as="h2" id="usage">
  使用方法
</Heading>
```

规则：

- `as` 可用 `h1` 到 `h6`。
- 优先使用 Markdown 标题；只有需要组件能力时才用 `Heading`。

### InlineTOC

用于文档内小目录。

```mdx
<InlineTOC
  items={[
    { title: "概览", url: "#overview", depth: 2 },
    { title: "使用", url: "#usage", depth: 2 }
  ]}
>
  本页目录
</InlineTOC>
```

规则：

- `items` 是静态数组。
- 每项必须有 `title`、`url`、`depth`。
- 不要动态生成目录。

### TypeTable

用于展示组件 props、API 字段、配置项。

```mdx
<TypeTable
  type={{
    title: {
      type: "string",
      description: "显示标题。",
      required: true
    },
    disabled: {
      type: "boolean",
      description: "是否禁用。",
      default: "false"
    }
  }}
/>
```

规则：

- `type` 必须是静态对象。
- 字段必须包含 `type`。
- `description` 建议简短。

### ImageZoom

用于可放大的图片。

```mdx
<ImageZoom src="https://placehold.co/640x360/png" alt="示例图片" width={640} height={360} />
```

规则：

- 必须写 `src` 和 `alt`。
- 建议写 `width` / `height`。
- 本地相对图片路径需等资源解析完善后再大量使用。

### DynamicCodeBlock

用于动态高亮代码块。

```mdx
<DynamicCodeBlock
  lang="ts"
  code={'export const message = "hello"'}
  codeblock={{ title: "example.ts" }}
/>
```

规则：

- 优先使用普通 fenced code block。
- 只有需要传 `title`、动态展示时才用。
- `code` 必须是静态字符串。

### GithubInfo

用于展示 GitHub 仓库信息。

```mdx
<GithubInfo owner="fuma-nama" repo="fumadocs" />
```

规则：

- 会发起网络请求；离线环境可能失败。
- 默认不推荐 AI 自动使用。
- 不要写 `token`。

## 临时兼容组件

暂无。未知组件会导致渲染失败，请不要使用。

## 禁止内容

禁止：

```mdx
import Something from './Something'

export const data = {}

function MyComponent() {
  return <div />
}

<MyUnknownComponent />

{fetch('/api')}

{window.location.href}
```

禁止原因：

- 破坏本地渲染稳定性。
- 可能触发安全问题。
- 组件不存在会导致渲染失败。

## AI 生成 MDX 的固定要求

AI 生成 MDX 时必须：

1. 输出完整 MDX。
2. 包含 frontmatter。
3. 只使用白名单组件。
4. 不写 import/export。
5. 不写自定义 React 组件。
6. 不写运行时代码。
7. 代码示例放进 fenced code block。
8. 如果需要强调信息，优先用 `Callout`。
9. 如果需要资源入口，使用 `Cards/Card`。
10. 如果不确定组件是否支持，退回普通 Markdown。

## 推荐模板

```mdx
---
title: 标题
description: 一句话描述
---

# 标题

开头说明。

<Callout title="提示">
这里放重要提示。
</Callout>

## 小节

正文内容。

```ts
console.log('example')
```

## 相关资源

<Cards>
  <Card title="资源名称" href="https://example.com">
    资源描述。
  </Card>
</Cards>
```
